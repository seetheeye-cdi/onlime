# 카카오톡 자동화 가이드

> 이 문서는 개발을 모르는 사람도 따라할 수 있도록 만들었습니다.
> 모르는 부분이 있으면 Claude에게 물어보세요!

## 이게 뭐하는 건가요?

한 줄 요약: **카카오톡 메시지를 자동으로 읽어서, 옵시디언 데일리노트에 정리해주는 시스템**

### 작동 원리 (그림으로 이해하기)

```
[S26 Ultra 폰]                    [맥북]                      [옵시디언]
     |                               |                            |
     |  1. 카톡 알림이 뜸             |                            |
     |  2. Termux가 알림을 캡처       |                            |
     |  3. 10초마다 서버로 전송 ------>|  4. 서버가 메시지 저장       |
     |                               |  5. 누가 보냈는지 파악       |
     |                               |  6. 데일리노트에 정리 ------>| 7. 오늘 카톡 요약!
     |                               |                            |
```

### 좀 더 자세히

1. **폰에서 카톡이 오면** 알림이 뜨죠? (알림센터에 "심정혁: 회의 시간 변경됐습니다" 이런 식으로)
2. **Termux라는 앱**이 그 알림을 읽습니다 (카톡 내용을 직접 보는 게 아니라, 알림만 읽는 거예요)
3. **10초마다** 새로운 알림이 있으면 맥북 서버로 보냅니다
4. **맥북의 Onlime 서버**가 받아서:
   - 중복 메시지 걸러내고
   - 카톡 닉네임을 실명으로 바꾸고 (예: "정혁이형" → "심정혁")
   - 채팅방별로 그룹화합니다
5. **데일리노트에 자동 삽입**: `## ==잡서` 섹션 아래에 오늘의 카톡 요약이 들어갑니다

### 데일리노트에 이렇게 나옵니다

```markdown
## ==잡서
#### 오늘의 일정
- 10:00~11:00 [[20260402_팀미팅_Meeting|팀미팅]]

<!-- SYNC:kakao-digest -->
#### 카카오톡 요약

**참치상사 팀방** (12건, 09:30~14:20)
- 참여자: 심정혁, 김욱영, 조연우
- 최근: "수고하세요" (조연우, 14:20)
- 최근: "네 알겠습니다" (김욱영, 14:18)
- 최근: "회의 시간 변경됐습니다" (심정혁, 09:30)

**이준석** (3건, 10:00~10:15)
- 최근: "내일 점심 같이 할래?" (이준석, 10:15)
<!-- /SYNC:kakao-digest -->
```

## 세팅 가이드 (순서대로 따라하세요)

### Step 1. API 키 만들기 (맥북에서)

API 키는 **비밀번호** 같은 거예요. 아무나 서버에 데이터를 보내지 못하게 막아줍니다.

**터미널을 열고** (Cmd + Space → "터미널" 검색):

```bash
# 랜덤 API 키 생성 (한 번만 하면 됩니다)
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```
XvVgpclsq_2uf3n3XSFOn2Vx_2QeLOC3Lu6gGvmQE28

이렇게 나오는 긴 문자열을 복사하세요. 예시:
```
aB3dE5fG7hJ9kL1mN3pQ5rS7tU9vW1xY3z
```

이제 이걸 `.env` 파일에 저장합니다:

```bash
# Onlime 폴더로 이동
cd ~/Desktop/Onlime

# .env 파일에 API 키 추가
echo 'ONLIME_API_KEY=여기에_위에서_복사한_키_붙여넣기' >> .env
```

> **주의**: `.env` 파일은 절대 다른 사람에게 보여주지 마세요!

### Step 2. 닉네임 매핑 — 거의 자동! (맥북에서)

카카오톡 닉네임을 실명으로 바꿔주는 기능인데, **대부분 자동으로 됩니다.**

시스템이 자동으로 읽는 데이터:
- `contacts.csv` — Google 연락처 (475명 추출됨)
- 옵시디언 `People` 폴더 — 373명의 인물 노트
- `onlime.toml`의 `email_to_name` — 참치상사 팀원 8명
- `onlime.toml`의 `known_contacts` — 주요 연락처 10명

**총 663명의 이름을 자동 인식합니다.**

자동 매칭 예시:
| 카톡 닉네임 | 자동 매칭 결과 | 매칭 방식 |
|------------|--------------|----------|
| 심정혁 | 심정혁 | 실명 그대로 |
| 정혁 | 심정혁 | 이름만으로 매칭 |
| 정혁이형 | 심정혁 | 존칭 제거 후 매칭 |
| 욱영이 | 김욱영 | 접미사 제거 후 매칭 |
| 준호형 | 윤준호 | 존칭 제거 후 매칭 |

> **하나도 안 건드려도 대부분 잘 됩니다!**

만약 자동 매칭이 안 되는 특이한 닉네임이 있다면, 그때만 `onlime.toml`에 추가하세요:

```toml
[kakao.nickname_to_name]
# 자동 매칭 안 되는 경우만 여기에 추가
# "카톡닉네임" = "실명"
# 예: "참치킹" = "최동인"
```

### Step 5. 폰에 Termux 설치 (S26 Ultra에서)

Termux는 안드로이드에서 리눅스 명령어를 쓸 수 있게 해주는 앱이에요.

1. **F-Droid 설치** (Google Play의 Termux는 오래됐어요)
   - 폰 브라우저에서 `https://f-droid.org` 접속
   - F-Droid APK 다운로드 & 설치
   - (보안 경고가 뜨면 "그래도 설치" 선택)

2. **F-Droid에서 설치할 앱 2개**:
   - `Termux` 검색 → 설치
   - `Termux:API` 검색 → 설치

3. **Termux:API에 알림 접근 권한 부여** (중요!)
   - 폰 설정 → 앱 → Termux:API → 권한
   - **알림 접근** 허용
   - 또는: 설정 → 알림 → 고급 → 알림 접근 허용 → Termux:API 켜기

---

### Step 6. Termux 초기 설정 (S26 Ultra Termux에서)

Termux 앱을 열고, 아래 명령어를 **한 줄씩** 입력하세요:

```bash
# 1. 패키지 업데이트
pkg update && pkg upgrade -y

# 2. 필요한 도구 설치
pkg install -y termux-api python

# 3. Python 패키지 설치
pip install requests

# 4. 설정 폴더 만들기
mkdir -p ~/.config/onlime
```

### Step 7. 캡처 스크립트 폰으로 복사

**방법 A: USB 케이블로 복사** (간단)
1. S26 Ultra를 맥북에 USB-C로 연결
2. Android File Transfer 또는 Finder에서 폰 저장소 열기
3. 맥북의 `~/Desktop/Onlime/scripts/termux_capture.py` 파일을
4. 폰의 `Download` 폴더에 복사

그 다음 Termux에서:
```bash
cp /storage/emulated/0/Download/termux_capture.py ~/termux_capture.py
```

**방법 B: 맥북에서 직접 전송** (명령어에 익숙하다면)
```bash
# 맥북 터미널에서 (새 탭으로):
# IP는 폰의 IP 주소 (폰 설정 → Wi-Fi → 연결된 네트워크 탭 → IP주소)
# 이 방법은 폰에 SSH가 설정된 경우에만 가능합니다
```

**방법 C: 직접 입력** (가장 확실)

Termux에서:
```bash
# 에디터로 파일 만들기
nano ~/termux_capture.py
```
그 다음 맥북에서 `scripts/termux_capture.py` 파일 내용을 전체 복사해서 Termux에 붙여넣기 → Ctrl+O → Enter → Ctrl+X

### Step 8. 캡처 스크립트 실행 (S26 Ultra Termux에서)

```bash
# API 키 설정 (Step 1에서 만든 키를 여기에 넣으세요)
export ONLIME_API_KEY="여기에_Step1에서_만든_키_붙여넣기"

# 실행! (192.168.0.XXX를 Step 3에서 확인한 맥북 IP로 바꾸세요)
python ~/termux_capture.py --server http://192.168.0.XXX:8000
```

## 자주 묻는 질문

### Q: 카톡 내용이 다 보이나요?
**A: 아니요, 알림에 나오는 내용만 보입니다.**
- 카카오톡 알림에 뜨는 미리보기 메시지만 캡처합니다
- 사진, 동영상, 이모티콘은 "(사진)", "(동영상)" 등으로 표시됩니다
- 알림을 안 읽고 쌓아두면 최신 메시지만 보일 수 있어요
- 대략 70~80% 정도의 메시지를 캡처할 수 있습니다

### Q: 보안은 괜찮은가요?
**A: 네, 이렇게 보호됩니다:**
- API 키가 있어야만 서버에 데이터를 보낼 수 있어요
- 폰과 맥북이 같은 Wi-Fi에 있을 때만 작동합니다
- 데이터는 외부 서버로 절대 안 나갑니다 (전부 내 맥북 안에서 처리)
- 카카오톡 계정에 로그인하는 게 아니라, 알림만 읽는 거예요

### Q: 배터리 많이 먹나요?
**A: 거의 안 먹어요.**
- 10초마다 알림 목록만 확인하는 가벼운 작업입니다
- Termux:API의 알림 접근은 시스템 레벨이라 배터리 소모가 적습니다

### Q: 폰 잠금 상태에서도 되나요?
**A: 네, Termux가 백그라운드에서 돌아갑니다.**
- 단, 배터리 최적화에서 Termux를 제외해야 합니다:
  - 설정 → 앱 → Termux → 배터리 → "제한 없음" 선택

### Q: 그룹채팅이랑 1:1 채팅 구분되나요?
**A: 네, 자동으로 구분됩니다.**
- 그룹채팅: 채팅방 이름 + 누가 보냈는지 표시
- 1:1 채팅: 상대방 이름으로 표시

### Q: 서버가 꺼져있을 때 온 카톡은?
**A: 폰이 다음 사이클에 다시 보내려고 시도합니다.**
- 서버가 꺼져있으면 "서버 연결 실패" 로그가 뜨고
- 10초 후 다시 시도합니다
- 단, 알림이 지워지기 전에 서버를 켜야 합니다

### Q: 매번 이렇게 실행해야 하나요?
**A: 자동 실행 설정을 하면 편합니다.**

맥북 서버 자동 시작 (터미널에서):
```bash
# 맥북 부팅 시 자동 실행하려면 LaunchAgent 설정이 필요합니다
# 이건 나중에 Claude에게 "서버 자동 시작 설정해줘"라고 요청하세요
```

Termux 자동 시작 (폰에서):
```bash
# Termux:Boot 앱을 F-Droid에서 추가 설치하면
# 폰 부팅 시 자동으로 캡처 스크립트를 실행할 수 있습니다
# ~/.termux/boot/ 폴더에 스크립트를 넣으면 됩니다
```

## 문제가 생겼을 때

### "서버 연결 실패" 에러
1. 맥북에서 서버가 켜져있는지 확인
2. 폰과 맥북이 같은 Wi-Fi인지 확인
3. IP 주소가 맞는지 확인: 맥북에서 `ipconfig getifaddr en0`
4. 방화벽 확인: 시스템 설정 → 네트워크 → 방화벽 → 8000 포트 허용

### "401 Unauthorized" 에러
- 폰의 API 키와 맥북 `.env`의 API 키가 같은지 확인
- 대소문자, 공백 주의!

### "termux-notification-list not found" 에러
- `Termux:API` 앱이 설치되어 있는지 확인
- `pkg install termux-api` 다시 실행

### 알림이 안 잡히는 경우
- 설정 → 알림 → 고급 → 알림 접근 허용 → Termux:API가 켜져있는지 확인
- 카카오톡 알림이 켜져있는지 확인 (설정 → 앱 → 카카오톡 → 알림)
