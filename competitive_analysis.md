# Onlime 경쟁 분석 보고서

> 작성일: 2026-04-02
> 목적: Onlime이 진입하는 시장의 기존 플레이어 분석 및 차별화 포인트 도출

---

## 1. Onlime이 하는 일 (우리 제품 정의)

Onlime은 **개인 AI 워크플로우 시스템**으로, 다음을 자동화한다:

1. **Google Calendar 동기화** -- 캘린더 이벤트를 Obsidian 미팅 노트로 자동 생성
2. **Plaud 녹음기 연동** -- 녹음 파일을 시간 기반으로 캘린더 이벤트에 자동 매칭
3. **AI 요약/녹취록 삽입** -- Plaud의 전사(transcription), AI 요약, 주제 아웃라인을 미팅 노트에 자동 삽입
4. **엔티티 자동 링킹** -- 본문에서 사람, 조직, 개념을 인식하여 `[[위키링크]]`로 자동 변환
5. **데일리 노트 요약** -- 하루의 녹음들을 종합하여 데일리 노트에 한 줄 요약 삽입
6. **인물 그래프 구축** -- 미팅 참석자를 frontmatter에 기록, 관계 네트워크 자동 형성
7. **한국어(English) 패턴 인식** -- `캔디드(Candid)` 같은 패턴을 자동 발견하여 엔티티 생성

핵심 차별점: **녹음 → 캘린더 매칭 → 노트 생성 → 엔티티 링킹 → 관계 그래프**를 하나의 파이프라인으로 연결하며, 모든 데이터가 로컬 Obsidian vault에 마크다운으로 저장된다.

- 전사 후 **지식 그래프 구축**이나 **인물 관계 자동 연결**을 해주는 서비스는 없음
- Onlime은 Plaud 하드웨어 녹음기를 활용하여 대면 미팅을 커버하는 유일한 파이프라인

| **강점** | 8,000+ 앱 연동, 미팅 라이프사이클 전체 자동화 가능 |

**핵심 차이:** Zapier/Make는 클라우드 서비스 간 연결에 특화되어 있지만, Onlime처럼 **로컬 Obsidian vault에 직접 마크다운을 쓰고 위키링크로 연결하는 것은 구조적으로 불가능**하다.

## 4. Onlime만이 채우는 시장 갭 (Gap Analysis)

### Gap 1: 대면 미팅 → 구조화된 지식 파이프라인
- **현황:** 화상회의 봇(Otter/Fireflies/tl;dv/Granola)은 대면 미팅을 커버하지 못함. Plaud/클로바노트는 녹음-전사까지만 하고 끝남.
- **Onlime의 해결:** Plaud 녹음 → 시간 기반 캘린더 매칭 → Obsidian 미팅 노트 → 엔티티 링킹까지 자동

### Gap 2: 전사 결과의 지식 그래프 통합
- **현황:** 어떤 도구도 전사 결과에서 사람/조직/개념을 자동 인식하여 기존 지식 베이스와 연결하지 못함
- **Onlime의 해결:** VaultIndex가 기존 vault의 위키링크를 스캔하고, auto_link가 새 녹취록에서 매칭되는 엔티티를 자동으로 `[[위키링크]]`로 변환. 한국어(English) 패턴까지 자동 발견

### Gap 3: 미팅 내용 기반 인물 관계 관리
- **현황:** Personal CRM(Monica/Clay/Dex)은 "누구를 만났는지"만 추적. "무슨 이야기를 했는지"는 수동 입력 필요
- **Onlime의 해결:** 미팅 노트 frontmatter에 participants를 자동 기록하고, Obsidian의 backlink로 인물별 미팅 이력을 자동 집계

### Gap 4: 데이터 주권 (로컬 마크다운)
- **현황:** Otter/Fireflies/클로바노트 모두 클라우드 종속. 서비스 종료 시 데이터 유실 위험
- **Onlime의 해결:** 모든 데이터가 로컬 Obsidian vault에 마크다운 파일로 저장. 영원히 이동/검색/백업 가능

### Gap 5: 한국어 비즈니스 컨텍스트
- **현황:** 글로벌 도구들은 한국 비즈니스 문화(카카오톡 소통, 한국어 이름 패턴, 한국 조직 구조)를 이해하지 못함
- **Onlime의 해결:** 한국 성씨 인식(`_KOREAN_SURNAMES`), 이메일→이름 매핑, known_contacts 관리, 한국어(English) 복합 엔티티 인식

### Gap 6: 하루 단위 통합 브리핑
- **현황:** 개별 미팅 노트는 만들어주지만, 하루의 모든 미팅을 종합한 요약을 데일리 노트에 삽입하는 도구는 없음
- **Onlime의 해결:** `build_daily_summary`가 하루의 모든 녹음을 수집하여 한 줄 요약 + 상세 노트 링크를 데일리 노트에 자동 삽입

## 7. 결론 및 권장사항

### 우리의 핵심 경쟁우위 (지켜야 할 것)
1. **Plaud → 캘린더 → Obsidian 자동 파이프라인** -- 이것을 구현한 도구는 세계적으로 없음
2. **엔티티 자동 링킹 + 지식 그래프** -- 녹취록에서 인물/조직을 인식하여 기존 지식과 연결
3. **로컬 데이터 소유** -- 마크다운 기반으로 벤더 종속 없음
4. **한국어 비즈니스 컨텍스트** -- 한국 성씨/이름 패턴, 한국어(English) 복합 엔티티

### 단기 강화 필요 영역 (Phase 2)
1. **AI 요약 프로세서** -- Plaud 요약에 의존하지 않고 자체 LLM 기반 요약 (summarizer.py 구현)
2. **인사이트 생성** -- 크로스 미팅 분석 (insight.py 구현)
3. **웹 UI 대시보드** -- CLI 전용에서 벗어나 직관적 모니터링 (web/ 이미 존재)

### 중기 확장 영역 (Phase 3-4)
1. **카카오톡/슬랙 커넥터** -- 미팅 외 커뮤니케이션 통합 (스텁 이미 존재)
2. **미팅 전 브리핑** -- 다음 미팅 참석자의 과거 미팅 요약 자동 생성
3. **관계 강도 대시보드** -- 인물별 미팅 빈도/최근성/주제 분석
4. **후속조치 추적** -- 액션 아이템 추출 및 완료 여부 추적
