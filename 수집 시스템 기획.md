[기획안] LLM 기반 개인 지식 베이스(Second Brain) 구축 아키텍처본 기획안은 산재된 개인의 정보, 대화, 아이디어를 하나의 저장소(Google Drive + Obsidian)로 통합하고, 이를 LLM이 학습 및 검색할 수 있는 형태(.md)로 자동 변환하는 데이터 파이프라인 구축을 목표로 합니다.

1. 시스템 핵심 구조 (Architecture)저장소 (Storage): 구글 드라이브(Google Drive for Desktop) 폴더 동기화프론트엔드 (UI): Obsidian (데이터 열람, 위키 연결, 시각화) 및 Telegram (통합 입력창/비서)백엔드/자동화 (Backend): Python 기반 전용 자동화 서버 (24시간 가동)데이터 포맷: 모든 데이터는 LLM과 Obsidian이 읽기 쉬운 Markdown(.md)으로 정규화

2. 소스별 데이터 수집 파이프라인 (Data Ingest)가장 피로도가 높았던 '화면 점유'와 '폐쇄형 플랫폼의 API 부재' 문제를 우회하여, 백그라운드 처리와 사용자 행동 기반의 수집 방식을 채택합니다.

A. 카카오톡 (자동화: Pywinauto)방식: Python pywinauto 라이브러리를 활용한 백그라운드 UI 제어.동작: 3시간마다 카카오톡 PC 버전의 '대화 내역 내보내기' 핸들(Handle)을 찾아 백그라운드에서 클릭 및 .csv 저장 $\rightarrow$ 스크립트가 대화 내용 파싱 및 요약 $\rightarrow$ .md 저장.장점: 마우스 포인터를 뺏기지 않고, 작업 흐름의 방해 없이 조용히 데이터를 적재합니다.

B. 모바일 네이티브 (삼성 S26 Ultra)방식: 서드파티 동기화 앱(AutoSync 등) 활용.동작: 폰으로 찍은 사진 및 기본 음성 녹음 파일이 구글 드라이브 특정 폴더로 자동 업로드 $\rightarrow$ 백엔드 서버가 이를 감지하여 Whisper STT로 텍스트 변환 및 .md 적재.

C. 웹 브라우저 & 유튜브방식: PC에서는 Obsidian Web Clipper, 모바일에서는 텔레그램 공유 기능 활용.동작: 유튜브 영상이나 웹 아티클을 텔레그램 봇으로 전송 $\rightarrow$ 서버가 백그라운드에서 자막(Transcript) 및 본문을 추출하고 LLM 요약을 거쳐 .md로 저장.

3. 통합 커맨드 센터: 텔레그램 봇 설계플랫폼(Gemini, Claude 등)의 폐쇄성을 극복하기 위해, '텔레그램 봇'을 유일한 정보 수집 채널이자 AI 비서 인터페이스로 구축합니다. (인사이트 북마크 방식)봇의 2가지 핵심 모드데이터 수집기 (Data Catcher):사용자가 링크, 텍스트, 음성 메시지를 봇에게 보내면, 봇이 데이터를 정제하여 Obsidian raw/ 폴더에 즉시 마크다운으로 저장합니다.지식 비서 (AI Assistant):일반적인 질문을 하면 Gemini 등 LLM API를 호출해 대답합니다. 필요시 내 Obsidian 폴더를 검색(RAG)하여 맞춤형 답변을 제공합니다.해시태그 기반 자동 라우팅 시스템봇에게 메시지나 링크를 보낼 때 해시태그를 포함하면, 백엔드 서버가 이를 인식해 지정된 프로젝트 폴더로 자동 분류합니다.

4. 기대 효과 및 워크플로우 요약입력의 단순화: 사용자는 정보의 출처(PC, 모바일, 웹)에 상관없이 의미 있는 정보를 텔레그램으로 던지기만 하면 됩니다.방해 없는 자동화: 카카오톡 등 로컬 데이터는 사용자의 마우스를 뺏지 않고 백그라운드에서 6시간마다 조용히 병합됩니다.지식의 자산화: LLM이 정기적으로 raw/ 폴더를 순회하며 위키 문서를 컴파일하고, 누락된 정보를 채우는 'Health Check'를 수행하여 완벽한 개인용 LLM OS를 완성합니다.
